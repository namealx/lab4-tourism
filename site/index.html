Lab 4 / Tourism Bureau

Console travel management with SQLite, tests, coverage, and docs.

The application manages buses, crew, routes, trips, orders, and reporting. GitHub Actions builds the project, uploads coverage to Codecov, and publishes this page through GitHub Pages.

Source code Codecov Wiki
C + SQLite Role-based menus SELECT reports Unit tests

What is included

Core flows that already work

  • Authentication with seeded roles for admin, manager, crew, and client.
  • CRUD for buses, crew, routes, and trips backed by SQLite tables.
  • Reporting queries for trips, salary calculations, routes, and mileage.

Local setup

How to run it

make
make check
make coverage
./bin/tourism_app

Coverage builds into a separate directory, so the normal build stays clean.

Automation

What GitHub Actions does

  • Builds and tests on Ubuntu and macOS.
  • Generates coverage and uploads it to Codecov with OIDC.
  • Deploys this static site to GitHub Pages from the main branch.
